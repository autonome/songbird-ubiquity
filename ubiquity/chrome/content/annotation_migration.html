Ubiquity Command Feed Migration

As of Ubiquity 0.1.3, we have changed the way that command feed subscriptions are tracked. This was done to significantly improve Ubiquity's performance and also allow command subscriptions to be kept when browser history is cleared (see bug #146 for more information).

Unfortunately, one side effect of this is that most of the command feeds you were subscribed to prior to this change are no longer being subscribed to.
