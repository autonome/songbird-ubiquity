Developing Ubiquity

Learn how to develop for Ubiquity by reading the command author tutorial.

Use the command editor to play around with creating new Ubiquity commands. This isn't just a fun way to experiment with Ubiquity—it's actually a really easy way to experiment with Firefox development in general, because writing and changing commands doesn't require restarting Firefox, or even manually reloading anything.

Get involved! Learn how by visiting the Ubiquity Wiki.

Share your commands. Post it on the commands in the wild wiki page. Also, sign up for the the herd command which will then display your command on the dashboard.

    Ubiquity 0.1.3 Released!

    Ubiquity 0.1.3 brings a range of new features to the board:

    • New Look: Ubiquity has a new, simpler look.
    • Skins: Anyone can now give Ubiquity a new look. It's as simple as a little bit of CSS. Once you've created a skin, it's easy to share it with the world: you can subscribe to skins just like you can subscribe to commands.
    • Speed: Ubiquity is now much more snappy.
    • Relevancy: Ubiquity makes it easier to issue the command you want, and the context menu is working again.

    If you're wondering where your old command feed subscriptions went, check out our handy Command Feed Migration page. Sorry for the inconvenience!

    Want to know more about what's changed? Read the release notes.

    Ubiquity is a team effort by contributors from all over the world. For a list of some of the contributors, check out the credits page.
